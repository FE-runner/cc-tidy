# App Router 路由

## 文件约定

| 文件 | 用途 |
|------|------|
| `page.tsx` | 路由 UI（路由可访问的必要条件） |
| `layout.tsx` | 共享布局（在子导航之间持久化） |
| `loading.tsx` | 路由段的 Suspense 回退 |
| `error.tsx` | 错误边界（需要 `'use client'`） |
| `not-found.tsx` | 404 UI（由 `notFound()` 触发） |

## 文件夹约定

| 模式 | 含义 |
|------|------|
| `(group)/` | 路由组 — 仅用于组织，不出现在 URL 中 |
| `_folder/` | 私有文件夹 — 共置代码，不是路由 |
| `[param]/` | 动态段 |
| `[...param]/` | 全捕获段 |
| `[[...param]]/` | 可选全捕获段 |

## 动态路由（Next.js 15）

`params` 是 `Promise` — 在解构之前始终 `await`。
`generateStaticParams()` 用于已知动态路由的 SSG。

## 并行路由和拦截路由

| 功能 | 用例 | 约定 |
|------|------|------|
| 并行路由 | 同时独立的 UI（仪表板面板、模态框） | 布局中的 `@slot/` 文件夹 |
| 拦截路由 | 软导航时显示为模态框，硬导航时显示完整页面 | `(.)`、`(..)`、`(...)` 前缀 |

务必为具有独立加载部分的仪表板使用并行路由。
务必为同时作为独立页面工作的照片/详情模态框使用拦截路由。

## 导航

- 务必使用 `next/link` 进行声明式导航（启用预获取）
- 务必仅在客户端组件中使用 `useRouter()` 进行编程式导航
- 务必在替换历史记录时使用 `router.replace()` 而非 `router.push()`（登录重定向）
- 禁止使用 `<a>` 标签进行内部导航 — 会丢失客户端路由优势

## 路由组

务必使用路由组 `(marketing)`、`(app)` 在不影响 URL 的情况下应用不同布局。
禁止不必要地嵌套路由组 — 保持结构扁平且可读。

## 反模式

- 禁止同步解构 `params` — 在 Next.js 15 中它是 Promise
- 禁止创建深层嵌套的路由组 — 增加复杂性而无价值
- 禁止在有慢速数据获取的路由中忘记 `loading.tsx` — 没有它用户会看到空白页面
