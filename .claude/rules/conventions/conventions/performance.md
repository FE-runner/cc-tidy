# 性能规则

## 数据库

- 防止 N+1 查询 — 使用预加载或批量加载，永远不要在循环中查询
- 在以下字段上添加索引：外键、WHERE 列、ORDER BY 列、JOIN 列
- 只查询需要的字段 — 生产代码中永远不要使用 `SELECT *`
- 对所有列表查询分页 — 永远不要返回无界结果集

## 缓存

- 对开销大的操作设置带 TTL 的缓存 — 写入时失效
- 对重复调用相同输入的纯函数进行记忆化
- 使用有界缓存（LRU）— 无界缓存就是内存泄漏

## 异步操作

- 并行运行独立操作（`Promise.all`、`asyncio.gather`、`Task.WhenAll`）
- 禁止在异步上下文中用同步计算阻塞事件循环
- 对大数据传输使用流式处理，而非全部加载到内存

## 内存

- 在销毁时清理订阅、事件监听器和定时器
- 限制集合大小 — 缓存使用 LRU 或环形缓冲区
- 注意持有大对象引用的闭包

## 快速决策矩阵

| 场景 | 操作 |
|------|------|
| 多个独立的 I/O 调用 | 并行运行 |
| 返回给客户端的大数据集 | 分页或流式传输 |
| 重复的高开销计算 | 记忆化或缓存 |
| UI 中 > 100 条目的列表 | 使用虚拟滚动 |
| 慢查询 | 分析、添加索引、检查 N+1 |
| 异步上下文中的重同步工作 | 卸载到 worker 或分块处理 |

## 反模式

- 禁止在未分析的情况下优化 — 先测量，再优化
- 禁止没有失效策略的缓存 — 过期数据导致 bug
- 禁止将整个表加载到内存 — 分页或流式处理
- 禁止创建没有清理逻辑的订阅
